form태그 활용하기

프론트(html페이지)에서 작성한 다수의 데이터를 서버에 전송할때 사용하는 태그
form태그는 데이터를 받을 수 있는 input태그를 자식태그로 사용
form태그는 속성으로 action, method를 필수적으로 설정한다.
action : 요청을 보낼 주소값을 설정
method : 데이터를 전송하는 방식을 설정(GET(default)/POST)

method속성
GET : 데이터를 url주소에 넣어서 전달하는 방식
전송되는 데이터에 제한이 있음
간단한 조회할때 사용, 게시글 조회, 회원 조회, 조건검색
*전송되는 데이터에 보안이 필요하다면 부적합함.
POST : 데이터를 request body에 넣어서 전달하는 방식
전송되는 데이터가 외부로 노출되지 않음, 전송되는 데이터에 제한이 없음 보안이 필요한 데이터를 전송하거나, 파일자체를 서버에 전송할때 사용

입력형식에 대해 알아보자

사용자에게 데이터를 입력받을때는 input태그를 이용한다.
input태그의 type속성에 따라 입력받는 UI가 달라진다.
일반적으로 input태그는 form태그의 자식태그로 사용함
type에 따라 value를 값을 고정으로 지정해줘야하는 input태그도 있다.
input태그의 내용은 서버에 key(name속성)=value형식으로 전달이됨
form태그와 같이 사용한 input태그는 반드시 name속성을 작성해줘야한다.

한문장을 입력받는 형식

type : test * 기본문자를 입력받을때 사용, 가장 많이 사용

type : password * 보안 필요한 문자를 입력받을때 사용

*

사용자에게 출력되지 않는 input 태그

type:hidden

클라이언트에게 보여주지 않지만 내부적으로 서버에서 필요한 데이터를 전송할때 사용 반드시 input내부에 name, value속성을 설정한다.
